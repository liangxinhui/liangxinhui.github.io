Linux top 持续监控脚本

发表于 2020-07-01

在定位 Linux 性能问题时，经常需要排查是哪个进程造成的系统资源（CPU、内存）过高。

这里提供一个脚本，可方便的进行持续的 top 监控，将监控结果按时间戳命名，保存到文件。

后续，可通过 grep 命令或脚本解析进行问题定位。

主要功能

  • 输出完整 top 进程列表
  • 支持自定义采集间隔
  • 支持删除过期日志
